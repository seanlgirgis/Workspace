LC #0242 — Valid Anagram Easy
HashMap / Counter Pattern

📊 The Problem

Given two strings `s` and `t`, return `true` if `t` is an anagram of `s`, and `false` otherwise.

Example: s = "anagram", t = "nagaram"

Goal: Determine effectively whether two literal sequences contain equivalent compositional characters ignoring index positioning.

💡 Key Insight

Two strings are mathematically guaranteed to be anagrams specifically when they exclusively consist of matching internal frequency boundaries exactly defining character repetition accurately universally without exceptions.

The Lego Box Disassembly:
If I break apart a castle and rebuild it into a spaceship, both objects exclusively use the exact identical collection of basic colored bricks without leaving spare parts remaining.

Why this works:

  • Comparing frequency arrays universally guarantees matching structures naturally bypassing geometric positional restrictions.
  • Python `collections.Counter` handles mapping instantly calculating hash limits directly scaling $O(N)$.

🔧 How the Algorithm Works

1 Quick Length Check

Return `False` instantly if the strings have different lengths. It is mathematically impossible for an anagram sequence to be successfully fulfilled if lengths miss match structurally natively checking bounds.

2 Frequency Counting

Use a HashMap (or Python's built-in `Counter`) to record the exact count of each character appearing in string `s` iterating exactly linearly tracking properties naturally representing parameters exactly mapping sequence attributes.

3 Equality Check

Compare the two instantiated frequency dictionaries `Counter(s) == Counter(t)` bounding equivalence properly avoiding linear traversal explicitly bypassing complex constraints.

🎯 The Frequency Counter Trick

Hash constraints effectively bypass structural variance tracking uniquely mapping quantities.

s: "rat"
t: "car"

Calculation:

• `Counter(s)` -> `{'r': 1, 'a': 1, 't': 1}`
• `Counter(t)` -> `{'c': 1, 'a': 1, 'r': 1}`
• Do they match? No. `t` has 'c', `s` has 't'.

Dictionaries successfully reduce comparison constraints linearly directly ignoring sequential positioning explicitly mapping quantitative structures functionally.

⚡ Why It's Optimal

❌ Sort Elements
Execute $O(N \log N)$ algorithm mapping sequential parameter limits sequentially sorting characters explicitly bypassing positional boundaries.

Time: O(n log n)
Space: O(n) (or O(1) in some languages)
✅ Counter Hash
Count occurrences mapping dictionaries completely bypassing sorted states natively representing identical quantitative mapping limits structurally resolving boundaries universally.

Time: O(n)
Space: O(1) (Bounded by 26 English letters)

Sorting takes algorithmic overhead. Space is bounded structurally to 26 limits successfully resolving parameter boundaries matching constraints mapping scaling effectively determining matching outputs structurally.
