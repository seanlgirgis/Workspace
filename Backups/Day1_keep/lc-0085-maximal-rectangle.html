LC #85 — Maximal Rectangle Hard
Monotonic Stack & DP Pattern

📊 The Problem

Given a 2D binary matrix filled with 0's and 1's, find the largest rectangle containing only 1's and return its area.

Example: matrix with histogram row [3, 0, 2, 5, 2]

Goal: Transform the 2D matrix into a series of 1D histograms and find the maximum area rectangle.

💡 Key Insight

Scanning all possible rectangles in a 2D matrix is extremely slow. Instead, treat each row of the matrix as the base of a 1D histogram. This lets us use the O(N) Monotonic Stack solution from LC 84 ("Largest Rectangle in Histogram").

The City Skyline:
Imagine building a city row by row from top to bottom. For each new row, if there's a building block (1), the tower grows taller. If there is an empty space (0), the entire tower above it crashes down to the ground. We can evaluate the skyline's maximum poster space at each step.

Why this works:

  • We maintain a running histogram of consecutive 1s looking upwards.
  • Updating the histogram takes only O(1) per cell (Dynamic Programming array).
  • Evaluating the histogram takes O(N) using a Monotonic Stack, giving an astonishing O(R × C) overall runtime.

🔧 How the Algorithm Works

1 Initialize Histogram Array

Create a 1D DP `histo` array of size `cols`, initially all 0s, to represent the height of 1s in the current row.

2 Top-Down Update

Iterate row by row. For each cell `(r, c)`:

  • Case 1: If `matrix[r][c] == "1"`, increase `histo[c]` by 1.
  • Case 0: If `matrix[r][c] == "0"`, reset `histo[c]` to 0 (the building lost its foundation).
3 Evaluate Histogram

Pass the updated `histo` array to the `largestRectangleArea` Monotonic Stack helper from LC 84.

4 Keep Global Maximum

Update `max_area` with the area found in this row. Return `max_area` after processing all rows.

🎯 The Monotonic Stack Trick

When we pop a bar, we know exactly how far it extends left and right.

Stack: [(idx=0, h=1), (idx=2, h=5)]
Current: idx=3, h=2 (drop in height!)

Stack Pop Calculation:

• Pop (idx=2, h=5). It was added at start_idx=2.
• It extends from idx=2 to idx=3 (exclusive).
• Area = 5 * (3 - 2) = 5

This guarantees we only calculate the area of a rectangle once it can no longer extend further right, achieving O(N) time for the histogram.

⚡ Why It's Optimal

❌ Brute Force
Check every possible top-left and bottom-right corner for a valid rectangle.

Time: O((R×C)²)
Space: O(1)
✅ DP + Monotonic Stack
Update a DP histogram per row and evaluate it using a stack.

Time: O(R×C)
Space: O(C)

Each cell is read once to update the histogram. In the stack, each bar is pushed and popped at most once per row, making the work per row strictly O(C), giving O(R×C) total time.

📖 Algorithm Explanation in Plain English

By looking at the matrix row by row from top to bottom, we can imagine the 1s forming vertical bars. When we see a 1, the bar at that column gets taller. When we see a 0, the bar crashes down to 0 because any rectangle coming from above is cut off. Every time we finish a row, we stop and ask: "What is the largest rectangle in this skyline?" We use a stack to quickly find the maximum area in that skyline before moving on to the next row.

🔑 The Confusing Part: Dropping to ZERO

It feels like resetting a cell's height to `0` destroys information about rectangles sitting higher up in the matrix.

Scenario:

Row 0: 1 1 1
Row 1: 1 0 1

Why This Matters: It's true the `0` cuts off the rectangle! But remember, we ALREADY calculated the maximum area of the `1 1 1` skyline when we processed Row 0. The global `max_area` variable safely stored that area (3) before Row 1 destroyed it.

Visual Analogy: Like taking a photo of the skyline at every step. If a building falls down, the old photo still exists showing its glorious max height.

💡 The Genius Move:
Decomposing a 2D matrix problem into R independent 1D array problems (using the Monotonic Stack), glued together by a lightning-fast DP state update.

📝 Annotated Solution

def maximalRectangle(matrix): if not matrix or not matrix[0]: return 0 # Helper function from LeetCode 84: Largest Rectangle in Histogram def largestRectangleArea(heights): max_area = 0 stack = [] # stores tuples of (start_index, height) for i, h in enumerate(heights): start = i # If we encounter a shorter bar, pop taller bars from stack while stack and stack[-1][1] > h: idx, val = stack.pop() # Calculate area for the popped bar max_area = max(max_area, val * (i - idx)) # The new shorter bar can extend back to where the taller bar started start = idx stack.append((start, h)) # Process any remaining bars in the stack for i, h in stack: max_area = max(max_area, h * (len(heights) - i)) return max_area max_max_area = 0 cols = len(matrix[0]) histo = [0] * cols # DP array to track histogram height # Top-down pass to build histograms and calculate area for row in matrix: for c in range(cols): if row[c] == "1": histo[c] += 1 else: histo[c] = 0 # A '0' cuts off any rectangle from above # Find max area for the current row's histogram max_max_area = max(max_max_area, largestRectangleArea(histo)) return max_max_area
Magic Line: histo[c] += 1 if row[c] == "1" else 0
Replacing the O(R) vertical scanning hook with a 1D DP array that updates in O(1) time per column.

🎤 Interview Q&A

Questions an interviewer will actually ask:

Q: What is the time complexity and why?
A: O(R × C). We visit each of the R rows and C columns once to update the `histo` array. The Monotonic Stack evaluates each row's histogram in O(C) time because each element is pushed and popped at most once. Total work is R × O(C) = O(R × C).
Q: What is the space complexity?
A: O(C). We map out a `histo` array of size C, and the `stack` inside the helper function can take at most O(C) space. We do not need a 2D matrix for DP.
Q: What happens if the matrix consists entirely of 0s?
A: The `histo` array remains all 0s. The stack helper loop processes it and calculates an area of 0, returning 0 efficiently.
Q: Why did you choose a Monotonic Stack instead of pure Dynamic Programming?
A: Treating the matrix as a series of histograms allows us to perfectly reuse the elegant stack mechanic from "Largest Rectangle in Histogram". A pure DP approach exists but has a more complicated state transition.
