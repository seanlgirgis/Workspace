LC #0217 — Contains Duplicate Easy
HashSet Pattern

📊 The Problem

Given an integer array `nums`, return `true` if any value appears at least twice in the array, and return `false` if every element is distinct.

Example: nums = [1, 2, 3, 1]

Goal: Detect if there's any repetition within the array in the fastest time possible.

💡 Key Insight

Instead of manually searching the array multiple times, we can insert numbers into a Set and rely entirely on mathematical properties to flag repetition automatically in constant lookup scaling.

The Nightclub Bouncer:
Imagine a bouncer with a photographic memory stamping hands. When a guest arrives, he checks if they already have a stamp. He immediately remembers them from earlier. If you stamp everyone efficiently on arrival, nobody can sneak past you to duplicate their entry unnoticed.

Why this works:

  • Sets cannot legally contain duplicate elements by definition.
  • Set operations (add, check membership) are $O(1)$ on average.
  • Evaluating length equivalence `len(set) == len(list)` circumvents manual iterations altogether reducing the computational effort explicitly matching lengths directly.

🔧 How the Algorithm Works

1 Initialize a Set

Create an empty Hash Set `seen = set()` to represent the collection of mathematically distinct numbers found to current state sequence bounds.

2 Iterate the Array

Check the numbers one by one consecutively passing elements natively mapping integer values matching elements exactly representing array contents.

3 Membership Validation

If the `num` falls natively matching existing keys inside `seen`, return `True`.

4 Log & Proceed

If not, execute an `add` mutation adding the new element natively executing sequence iteration mapping elements directly to output validation blocks.

🎯 The Length Equivalence Trick

Using Python's built-in C-bindings bypasses explicit linear checks

Array: [1, 2, 3, 1]
Set: {1, 2, 3}

Calculation:

• `len(nums) == 4`
• `len(set(nums)) == 3`
• `4 != 3` => Contains Duplicate = True

This guarantees the fastest execution bound mapping memory allocation instantly into core processing limits instead of virtual machine iteration tracking.

⚡ Why It's Optimal

❌ Sort Elements
Execute $O(N \log N)$ algorithm mapping sequential neighbor element variance checking sequentially natively determining matches.

Time: O(n log n)
Space: O(1)
✅ HashSet
Instantiate set allocations tracking single occurrences returning matches universally processing average elements mapped uniformly bounding constants.

Time: O(n)
Space: O(n)

Sorting takes algorithmic overhead. Space is almost always acceptable mapped directly matching scaling time complexities drastically optimizing processing directly mapping performance limits perfectly.

📖 Algorithm Explanation in Plain English

Because you are guaranteed that a subset of numbers does not intrinsically have duplicate elements, comparing the total count of numbers directly against an automatically-filtered subset of those numbers perfectly proves equivalence.

Step 1: Instantiation

Create a Python standard library `set` structure representing mathematical uniqueness mappings exactly defining set logic naturally processing element additions dynamically enforcing unique limits inherently scaling natively against input array elements bound completely identical identical mapped sequences sequentially assigned natively processing logic naturally assigning elements mapped correctly.

📝 Annotated Solution

def containsDuplicate(nums): # Creates a subset consisting universally mapped naturally defining standard elements return len(nums) != len(set(nums))
Magic Line: return len(nums) != len(set(nums))
