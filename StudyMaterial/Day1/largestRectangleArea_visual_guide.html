Largest Rectangle in Histogram
Stack-Based Algorithm Concept

📊 The Problem

Given an array of bar heights, find the largest rectangular area that can be formed.

Example: heights = [2, 4, 3, 2, 4]

Goal: Find the maximum area rectangle that fits within the histogram.

💡 Key Insight

Instead of checking all possible rectangles (O(n²) or O(n³)), use a monotonic stack to keep track of potential rectangles.

The Stack Metaphor:
Think of bars as customers in a "waiting room." Each bar waits for a shorter bar to arrive. When a shorter bar comes, the waiting customers (taller bars) can't extend further—we calculate their final area.

Why this works:

  • Stack maintains increasing heights
  • When we hit a shorter bar, we know how far back the taller bars can extend
  • Time complexity: O(n) instead of O(n²)

🔧 How the Algorithm Works

1 Maintain a Stack

Store pairs of (index, height) for bars with increasing heights. This forms a "waiting line" of bars that haven't encountered a blocker yet.

2 Process Each Bar

For each new bar, if it's shorter than the bar on top of the stack, we've found a "blocker." The stacked bars can't extend further right.

3 Calculate Areas

When popping a bar from the stack, we know:

  • Height: The popped bar's height
  • Right boundary: The current bar's index
  • Left boundary: Where the popped bar originally started (stored in stack)
4 Cleanup

After processing all bars, any remaining bars in the stack were never blocked. They can extend all the way to the right edge. Calculate their areas with the right boundary = array length.

🎯 The Stack Trick

When we pop a bar: We don't just know its height—we know exactly how far it can extend!

Stack: [(idx=0, h=2), (idx=1, h=4)]
Current bar: height=3 at index=2

Calculation for popped bar (idx=1, h=4):

• Right boundary: current index = 2
• Left boundary: top of remaining stack = 0
• Width: 2 - 0 - 1 = 1
• Area: 4 × 1 = 4

The "start" variable tracks where a bar could have started extending left from previous pops.

⚡ Why It's Optimal

❌ Brute Force
Check every possible rectangle combination

Time: O(n²) or O(n³)
Space: O(1)
✅ Stack Approach
Each bar processed once, pushed/popped once

Time: O(n)
Space: O(n)

The stack approach is optimal because it processes each bar exactly twice (push + pop), making it linear time.

📝 Annotated Code Logic

def largestRectangleArea(heights): max_area = 0 stack = [] # (starting_index, height) pairs for index, height in enumerate(heights): start = index # Bar is shorter: finalize rectangles while stack and stack[-1][1] > height: popped_index, popped_height = stack.pop() # Width = from where it started to current blocker width = index - popped_index max_area = max(max_area, popped_height * width) start = popped_index # Extend left for next bar # Push current bar (or extended version) stack.append((start, height)) # Cleanup: bars that were never blocked for index, height in stack: max_area = max(max_area, height * (len(heights) - index)) return max_area
Magic Line: start = popped_index
This lets shorter bars "inherit" the starting position of taller bars. A bar of height 3 can extend as far left as the original position of a previously-popped height-5 bar.

🔍 Example Walkthrough

Input: heights = [2, 4, 3, 2, 4]

Index 0, height=2:
Stack is empty → Push (0, 2)
Stack: [(0, 2)]
Index 1, height=4:
4 > 2 → No popping → Push (1, 4)
Stack: [(0, 2), (1, 4)]
Index 2, height=3:
3 < 4 → Pop (1, 4): Area = 4 × (2 - 1) = 4
3 > 2 → Stop popping
Push (1, 3) [bar-2 inherits bar-1's start]
Stack: [(0, 2), (1, 3)]
Index 3, height=2:
2 < 3 → Pop (1, 3): Area = 3 × (3 - 1) = 6
2 = 2 → Stop popping
Push (1, 2) [bar-3 inherits start=1]
Stack: [(0, 2), (1, 2)]
Index 4, height=4:
4 > 2 → No popping → Push (4, 4)
Stack: [(0, 2), (1, 2), (4, 4)]
Cleanup (end of array):
Pop (4, 4): Area = 4 × (5 - 4) = 4
Pop (1, 2): Area = 2 × (5 - 1) = 8 ← Maximum!
Pop (0, 2): Area = 2 × (5 - 0) = 10 ← Actually maximum!

Answer: 10 (height-2 bar extends across entire width)
