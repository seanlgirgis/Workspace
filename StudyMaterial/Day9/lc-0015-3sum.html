📋 Problem + Core Insight

🎯 LeetCode 15: 3Sum

The Core Insight: Fix one element with a for loop. Then solve Two Sum

for the remaining two elements with two pointers. By sorting the array first,

you can cleanly skip duplicate values and maintain an internal structure to easily converge on 0.

📋 Problem

Given integer array nums, return all triplets [nums[i], nums[j], nums[k]] where

i != j != k and nums[i] + nums[j] + nums[k] == 0. No duplicate triplets.

[-1, 0, 1, 2, -1, -4]
→ [[-1,-1,2], [-1,0,1]]

Constraints: 3 ≤ nums.length ≤ 3000 | -10⁵ ≤ nums[i] ≤ 10⁵

📊 Complexity Analysis

🔬 Complexity Analysis: Brute Force vs Optimal

The Core Insight: A naive three-loop approach takes O(n³).

By sorting the array (O(n log n)), we can fix the first number and use a linear O(n) scan for the remaining two. This bounds total time to O(n²).

Approach Time Space Notes
Brute force O(n³) O(1) Triple nested loops
Hash Map + Set O(n²) O(n) Deduplicating takes extra space
Two Pointers + Sort O(n²) O(1) Sort space is O(1) or O(log n)

🚀 Optimal Solution

def threeSum(nums: list[int]) -> list[list[int]]:
    nums.sort()
    result = []

    for i in range(len(nums) - 2):
        # Skip duplicate values for the fixed element
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        if nums[i] > 0:      # sorted: if nums[i] > 0, no triplet sums to 0
            break

        left, right = i + 1, len(nums) - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total == 0:
                result.append([nums[i], nums[left], nums[right]])
                # Skip duplicates for left and right
                while left < right and nums[left] == nums[left + 1]:  left += 1
                while left < right and nums[right] == nums[right - 1]: right -= 1
                left += 1
                right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1

    return result

🎤 Interview Q&A

🎤 5 Interview Q&A

Q1: Why sort first?

Answer: Sorting lets us use two pointers (sorted structure) AND skip duplicates easily (equal adjacent values).

Q2: Why O(n²) and not O(n)?

Answer: The outer loop is O(n); the inner two-pointer scan is O(n). So O(n) × O(n) = O(n²).

Cannot do better without hashing which increases space.

Q3: What is the "early exit" optimization?

Answer: if nums[i] > 0: break — if the smallest remaining element is positive,

the sum can only be positive. No valid triplet possible.

Q4: How does the algorithm avoid returning duplicate triplets?

Answer: It skips duplicates at three points: when choosing the first fixed element nums[i],

and when moving the left and right pointers after finding a valid triplet by checking

if the next element equals the previous element.

Q5: What happens if all elements in the array are zeroes?

Answer: The array is sorted to [0,0,0,...]. The first iteration finds `0+0+0 = 0`, adds `[0,0,0]`,

then skips all the remaining zeroes using the duplicate skipping logic, returning exactly one `[0,0,0]`.
