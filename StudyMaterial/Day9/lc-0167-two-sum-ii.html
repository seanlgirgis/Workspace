📋 Problem + Core Insight

🎯 LeetCode 167: Two Sum II (Input Array is Sorted)

The Core Insight: Two pointers is the pattern for problems on sorted arrays

where you need pairs. The trick: sorting creates structure. From a sorted array,

you can move pointers intelligently — left moves right to increase sum, right moves left to decrease.

📋 Problem

Given a sorted array numbers (1-indexed), find two numbers that add to target.

Return [index1, index2] (1-indexed). Use only O(1) extra space.

numbers = [2, 7, 11, 15]
target = 9
→ [1, 2]

Constraints: 2 ≤ numbers.length ≤ 3 * 10⁴ | -1000 ≤ numbers[i] ≤ 1000

📊 Complexity Analysis

🔬 Complexity Analysis: Brute Force vs Optimal

The Core Insight: A sorted structure eliminates the need to look back.

Instead of O(n) space for a HashMap, we use two variables for pointers getting O(1) space.

Approach Time Space Notes
Brute force O(n²) O(1) Nested loops
Hash Map O(n) O(n) Suboptimal memory
Two Pointers O(n) O(1) Leverages sorted order

🚀 Optimal Solution

def twoSum(numbers: list[int], target: int) -> list[int]:
    """
    Optimal Two Pointers
    Time: O(n)  |  Space: O(1)
    """
    left, right = 0, len(numbers) - 1

    while left < right:
        total = numbers[left] + numbers[right]
        if total == target:
            return [left + 1, right + 1]    # 1-indexed
        elif total < target:
            left += 1    # need bigger sum → move left right
        else:
            right -= 1   # need smaller sum → move right left

    return []   # guaranteed to have solution per problem

🎤 Interview Q&A

🎤 5 Interview Q&A

Q1: Why does this work?

Answer: The array is sorted. If sum < target, the only option to increase the sum is to use

a larger value, so we move the left pointer to the right. If sum > target, we need a smaller value,

so we move the right pointer to the left. We never skip a valid pair.

Q2: Why is this O(n) instead of O(n²)?

Answer: Each pointer moves at most n steps total. There are no nested loops — every

iteration eliminates at least one element from consideration. The loop runs at most n/2 times.

Q3: What if the array has duplicates?

Answer: The problem states there is exactly one solution. If duplicates existed and we needed

all unique pairs, we would just loop while numbers[left] == numbers[left - 1] to skip duplicates

after finding a match, maintaining O(n) runtime.

Q4: Can we use a HashMap instead of Two Pointers?

Answer: Yes, the standard Two Sum O(n) time/space works perfectly fine here.

However, the Two Pointers approach achieves O(1) space, making it strictly better

for already sorted arrays. We want to avoid allocating extra memory when possible.

Q5: What happens if the array is not sorted?

Answer: The Two Pointers approach relies on the monotonic property of the array.

If unsorted, we would have to sort it (O(n log n) time) or fallback to a HashMap

(O(n) time, O(n) space). Sorting destroys the original relative indices unless stored.
