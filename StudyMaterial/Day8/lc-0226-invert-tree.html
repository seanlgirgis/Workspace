📋 Problem + Core Insight

🌳 LeetCode 226: Invert Binary Tree

The Core Insight: At every node, swap its left and right children.

Then recursively invert both subtrees. The swap at each node is independent —

children can be inverted in any order. 4 lines total.

📋 Problem

Mirror a binary tree. Return its root.

Input:       Output:

   / \          / \
  2   7        7   2
 / \ / \      / \ / \
1  3 6  9    9  6 3  1

🧠 Mental Models

🧠 Choose Your Mental Model

Model The "Story" Mechanism
Mirror Flip "Place a vertical mirror at the root. Every node's left child becomes its right child and vice versa. This transformation applies at every node simultaneously." Swap children, recurse both subtrees
DFS Post-Order "Visit left, visit right, THEN swap. After both subtrees are inverted, swap the root's children." Post-order: process children before action at node
DFS Pre-Order "Swap first, then visit children. The swap at the root doesn't affect the recursion into children — they get correctly inverted regardless." Pre-order: swap at node, then recurse
BFS Level-by-Level "Go floor by floor. On each floor, swap every node's left and right children." Queue, swap children on dequeue

📊 Complexity Analysis

🔬 Complexity Analysis

The Core Insight: Every approach is O(n) time — we must visit every node.

Space differs: DFS uses call stack O(h), BFS uses queue O(w).

For this problem, both are equivalent — choose based on style preference.

Approach Time Space Style
DFS Recursive O(n) O(h) Most concise — 4 lines
BFS Iterative O(n) O(w) Explicit, no recursion limit concern
DFS Iterative (stack) O(n) O(h) Manual stack, avoids Python recursion

Why O(n) is the lower bound

You must visit every node to swap its children.

A node you don't visit won't get its children swapped — producing an incorrect result.

Therefore no algorithm can do better than O(n).

Space: h vs w

For a balanced binary tree: h ≈ log n, w ≈ n/2 (leaf level).

So DFS (O(log n)) is better than BFS (O(n)) for balanced trees.

For a skewed tree: h = n, w = 1. BFS (O(1)) beats DFS (O(n)).

✅ Default choice: DFS recursive — 4 lines, natural for trees.

Fallback to BFS if recursion depth is a concern (skewed tree with n > 1000).

🚀 Optimal Solution

def invertTree(root: TreeNode) -> TreeNode:
    """
    DFS Recursive — the canonical solution.
    Time: O(n)  |  Space: O(h) — call stack
    4 lines. The swap before recursion is fine because Python evaluates
    the right-hand side first: root.right, root.left = root.left, root.right
    is computed as a tuple swap, then assigned.
    """
    if not root:
        return None

    root.left, root.right = root.right, root.left   # swap

    invertTree(root.left)    # recurse into (previously right) subtree
    invertTree(root.right)   # recurse into (previously left) subtree

    return root


def invertTree_iterative(root: TreeNode) -> TreeNode:
    """
    DFS Iterative using explicit stack — avoids Python recursion limit.
    Time: O(n)  |  Space: O(h)
    """
    if not root:
        return None
    stack = [root]
    while stack:
        node = stack.pop()
        node.left, node.right = node.right, node.left
        if node.left:  stack.append(node.left)
        if node.right: stack.append(node.right)
    return root

🎤 Interview Q&A

🎤 5 Interview Q&A

Q1: Does the order of the recursive calls matter (left then right vs right then left)?

Answer: No. Both calls independently invert their subtrees. When we do invertTree(root.left),

we've already swapped, so root.left points to what was previously root.right.

Both subtrees get inverted regardless of call order. The result is identical.

Q2: Can you do this iteratively? Walk me through the stack approach.

Answer: Yes — use an explicit stack instead of the call stack:

stack = [root]
while stack:
    node = stack.pop()
    node.left, node.right = node.right, node.left
    if node.left:  stack.append(node.left)
    if node.right: stack.append(node.right)

This is DFS with an explicit stack. For BFS, use a queue and popleft() instead of pop().

Both produce a correctly inverted tree.

Q3: What is the space complexity for a perfectly balanced tree vs a skewed tree?

Answer: For DFS recursive, space = O(h) = call stack depth.

  • Balanced tree: h = log₂n → O(log n) space
  • Skewed tree (all nodes go right): h = n → O(n) space

For BFS: space = O(w) = max queue size = max level width.

  • Balanced tree: widest level has n/2 nodes → O(n) space
  • Skewed tree: widest level has 1 node → O(1) space

So DFS wins for balanced trees; BFS wins for skewed trees.

Q4: If you invert a tree twice, do you get the original back?

Answer: Yes — inverting twice is a self-inverse operation (like XOR with the same value).

invert(invert(tree)) == original. Proof: inverting swaps left↔right at every node.

Inverting again swaps right↔left (restoring original). The operation is its own inverse.

Q5: What real-world problem does inverting a tree model?

Answer: Mirror transformations in graphics (flip a scene graph left↔right).

Reversing precedence in decision trees (swap which branch is "preferred").

At Citi: inverting a region priority tree — if the primary region fails, we want the

secondary region to take priority, which can be modeled as inverting the tree and

processing it the same way.

🎯 Summary

🎯 Summary: Key Takeaways

The Pattern

Swap at every node, recurse into both children. Pre-order or post-order — both work.

The 4-Line Solution

def invertTree(root):
    if not root: return None
    root.left, root.right = root.right, root.left
    invertTree(root.left)
    invertTree(root.right)
    return root
