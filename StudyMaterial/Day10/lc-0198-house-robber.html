💰 LeetCode 198: House Robber

The Core Insight: At any given house `i`, you have a localized choice: either rob it (adding its value, but forfeiting house `i-1`), OR skip it entirely (keeping whatever max profits you accrued all the way through house `i-1`).

Structurally: max_profit(i) = max(rob_current + max_profit(i-2), skip_current_assume_i-1).

📋 Problem

You are planning to rob houses along a street, but adjacent houses carry connected security systems.

Return the maximum amount you can rob without triggering alarms natively exactly identically strictly sequentially.

Input: nums = [2,7,9,3,1]
Output: 12
Explanation: Rob house 1 (money = 2), rob house 3 (money = 9) and rob house 5 (money = 1).
Total amount you can rob = 2 + 9 + 1 = 12.

🚀 Optimal Solution

def rob(nums: list[int]) -> int:
    """
    Optimal Dynamic Programming — Constant Space Explicit Window.
    Time: O(n)  |  Space: O(1)
    """
    rob1, rob2 = 0, 0
    
    # [rob1, rob2, n, n+1, ...] moving the window forward
    for n in nums:
        temp = max(n + rob1, rob2)
        rob1 = rob2
        rob2 = temp
        
    return rob2

# Test
print(rob([2, 7, 9, 3, 1]))   # 12 -> 2 + 9 + 1
